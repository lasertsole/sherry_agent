---
Subagent 经验知识图谱闭环 — 实施方案书（v2）
一、背景与目标
1.1 现状问题
1. 无跨 session 经验复用：每次 subagent 任务从零开始
2. 压缩丢知识：对话压缩时被丢弃的关键发现永久丢失
3. 无主动提炼机制：Agent 执行中发现的有效策略和障碍没有被记录
1.2 目标
构建闭环：draft(主动记录) → distill(蒸馏) → ingest(写入) → recall(召回) → assemble(注入)
1.3 现有基础设施
组件	文件
多实例 DB	xp_graph/store/db.py
CRUD + FTS5 + 向量搜索	xp_graph/store/core.py
双路召回 + PPR	xp_graph/recaller/core.py
PageRank / 社区 / 去重 / 维护	xp_graph/graph/
XML 组装 + prompt 引导	xp_graph/format/assemble.py
对话知识提取	xp_graph/extractor/core.py
策略级 + 操作级蒸馏	xp_graph/extractor/distiller.py
多实例工厂	xp_graph/core.py
关键问题：distiller.py 引用了不存在的 agent.tools.subagent.draft，且形成了 xp_graph(底层) → subagent(上层) 的反向依赖。

2.2 模块分层与依赖方向
agent/tools/pub_base/xp_graph/       # 底层：纯基础设施，无业务依赖
  ├── store/      (DB CRUD)
  ├── recaller/   (召回)
  ├── graph/      (PageRank, 社区, 去重, 维护)
  ├── format/     (组装 XML)
  ├── extractor/  (对话知识提取 core.py，distiller.py 已移出)
  ├── type.py
  ├── core.py     (XpGraphInstance, after_turn, ingest_experiences)
  └── ...
         ↑ 被引用
         │

  ├── draft.py     # 草稿工具（@tool + get/clear/append）
  ├── distiller.py # 蒸馏（从 xp_graph 移出，引用 draft + xp_graph）
  └── ...
依赖方向单向：
- subagent/distiller.py → subagent/draft.py ✅ 同层
- subagent/distiller.py → xp_graph ✅ 上层 → 底层
- xp_graph → 无 subagent 依赖 ✅ 纯净
2.3 关键设计决策
决策项	选择
草稿工具形式	@tool 函数，agent 主动调用
注入方式	AIMessage，徊...徊 包裹
注入位置	第一个 HumanMessage 之后
蒸馏时机	方案 C：publish → distill → clear
distiller 位置	移出 xp_graph，放 subagent/
draft 位置	subagent/draft.py
Worker 草稿	finally 中合并到 Commander session
Fork 压缩前提炼	延后实施

5b. 方案 C 顺序调整：
async def _run_subagent(self, session_id, task_id, task, label):
    commander_session_id = f"commander-{session_id}"
    structured_response: SubAgentOutput | None = None
    # 知识图谱注入（5a）
    messages = [HumanMessage(content=task)]
    ...  # recall + assemble
    try:
        agent = build_commander()
        agent_res = await agent.ainvoke(
            input={..., "messages": messages}, ...
        )
        structured_response = agent_res.get("structured_response")
        announce_content = render_template_file(...)
    except HeartbeatTimeoutError as e:
        announce_content = render_template_file(...)
    except Exception as e:
        announce_content = render_template_file(...)
    # 方案 C：先发布 → 再蒸馏 → 最后清理
    msg = InboundMessage(...)
    await self._bus.publish_inbound(msg)         # ① 先发布
    try:
        result_text = structured_response.result if structured_response else ""
        await distill_and_ingest(                 # ② 再蒸馏
            task=task, result=result_text,
            session_id=session_id,
            commander_session_id=commander_session_id,
        )
    except Exception as e:
        logger.warning("[subagent] distillation failed: {}", e)
    Register.clear_all_register_sessions(commander_session_id)  # ③ 最后清理

七、风险与缓解
风险	缓解
draft 滥用产生噪音	蒸馏 prompt 强调 "Only extract genuinely reusable knowledge"；dedup_threshold=0.90
注入增加 token 消耗	recall_max_nodes=6，无召回结果则不注入
Worker 草稿合并竞态	SQLite 事务保护；Worker finally 中串行执行
蒸馏失败	try/except 保护，不影响主流程；json_mode 降级
召回不相关经验	Reranker 阈值 gap_score=0.5；PPR 降权低质量节点
方案 C 蒸馏延迟	auxiliary_llm 通常 <3s；超时有 except 保护
distiller 迁移后遗漏引用	全局搜索 from.*xp_graph.*distill 确认无残留
---
八、延后事项
1. 压缩前 Fork 提炼：核心闭环跑通后评估，风险最高
2. 边的蒸馏写入：当前 distill_and_ingest 只 ingest nodes，edges 被忽略，后续补充
