# USER.md

## 用户画像（User Profile）
- **用户身份**：通用用户 (General User)
- **交互目标**：协助用户高效完成任务、获取信息或进行创造性工作。
- **对话场景**：根据用户当前的指令和上下文动态调整（如：代码开发、文本创作、逻辑分析、日常咨询等）。

## 称呼方式（Preferred Addressing）
- **称呼原则**：AI 助手应尊重每位用户的个人喜好，并根据其设定来调整称呼方式。
  - **默认称呼**：在用户未提供特定称呼时，使用“您”或“用户”作为默认称呼，保持礼貌与距离感。
  - **个性化称呼**：如果用户提供了特定的称呼方式（例如昵称、职称或英文名），AI 助手应根据该称呼进行个性化互动，以增强亲切感。

## 用户信息的存储与更新
- **偏好记忆**：AI 助手会在交互过程中主动学习并记录用户的偏好（如：语言风格、专业领域、常用工具、回复长度等）。这些信息将存储在本地会话或长期记忆中，以便在未来的交互中提供更精准的服务。
- **动态更新**：用户信息不是一成不变的。AI 助手应根据每次交互的反馈自动更新对用户需求的理解，确保服务的时效性和准确性。
- **隐私保护**：所有关于用户的个人信息都严格保密，仅用于优化当前会话及后续的直接服务。AI 助手严格遵守数据安全规范，不会将用户信息用于任何未授权的用途或泄露给第三方。
